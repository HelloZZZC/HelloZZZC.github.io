单例模式(singleton)

单例模式用自己的话说就是在程序调用的过程中获取的对象实例只有一个。就好比通过getInstance()获取的对象实例的编号是#1，则在那之后所有通过调用getInstance()获取的对象实例的编号都是#1。

今天还看到了java的实现依靠的是内部类来控制实例化对象单一。实现如下:
java实现

与php与java中相同的是，都是通过私有化构造函数，防止new Class()创建实例，通过暴露出静态方法getInstance()来获取单例。与java实现不同的是，php还需要私有化__clone()和__wakeup()来防止生成新的对象。
private function __clone()

这表明通过clone创建了一个新的实例，因此我们需要私有化__clone()。私有化__clone()后再次通过同样的方法获取newSingletonObj时php会报出警告。
private function __wakeup()

这表明通过反序列化创建了一个新的实例，因此我们需要私有化__wakeup()。私有化__wakeup()后再次通过同样的方法获取newSingletonObj时php会报出警告。
